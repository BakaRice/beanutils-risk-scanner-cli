# BeanUtils 多页 HTML 报告设计

## 目标

把当前“列表 + 右侧详情面板”的单页报告改为真正的离线多页报告。主页面用于全局筛选与定位；点击任意扫描结果后，浏览器打开该调用点自己的详情页面。详情页面必须分别完整列出 Source Bean 和 Target Bean 的全部已解析 JavaBean 属性，属性是否存在同名映射不能影响其展示。

## 输出结构

假设 CLI 参数为 `--output target/report.html`，生成：

```text
target/report.html
target/report-details/
  finding-0001.html
  finding-0002.html
  ...
```

主页面和详情页都不依赖网络资源，可整体复制或压缩分享。详情目录名称由主报告文件名派生，避免同一目录下的多份报告互相覆盖。

## 主页面

- 顶部保留总数与 RISK、REVIEW、IGNORED、SAFE 统计。
- 保留全文搜索、Module 和结论筛选。
- 表格第一列继续是结论，随后是代码位置、Source 类型、Target 类型和属性概览。
- 每一整行都是可通过鼠标或键盘访问的详情链接。
- 删除右侧详情面板，给列表更完整的横向空间。

## 详情页面

- 顶部展示结论、代码位置、调用 Module 和调用形式。
- 提供返回列表、上一个调用、下一个调用导航。
- 展示原始调用代码以及 Source/Target 类型来源。
- `Source Bean 全部属性`表展示除 `TARGET_ONLY` 外的每一项。
- `Target Bean 全部属性`表展示除 `SOURCE_ONLY` 外的每一项。
- 两张表分别给出属性名、解析类型、声明类、映射状态和当前风险判定；父类属性与独有属性均不能省略。
- 另设`属性映射对照`表展示完整属性并集，方便逐项核对两侧对应关系与 Spring 5.3.1 判定原因。
- 未能解析类型的 REVIEW 调用明确展示诊断提示，不制造“没有属性”的结论。

## 兼容性与错误处理

- JSON 报告格式和扫描结论不变。
- 输出前创建详情目录；同名详情文件会被本次扫描结果覆盖，多出的旧详情文件会被清理，避免打开过期记录。
- HTML、属性和 JSON 均继续转义，防止源码内容破坏页面。

## 验证

- 报告单元测试验证主页面不再包含详情侧栏、每行指向独立详情页、详情文件数量正确。
- 单元测试验证 Source 独有、Target 独有、同名不可复制和继承属性分别进入正确的全量属性表。
- 运行全部 Maven 测试。
- 扫描真实 Demo，检查生成文件数量、链接目标、Source/Target 属性计数和零属性调用分类。
