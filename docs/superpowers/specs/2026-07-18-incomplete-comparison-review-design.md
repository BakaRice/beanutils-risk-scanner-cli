# 不完整属性对比必须待审查

## 目标

扫描器必须区分“完整分析后确认安全”和“无法完整探查但暂未发现风险”。后者不得返回 `SAFE`，必须返回 `REVIEW`。

## 完整性定义

Source 和 Target 分别生成一个属性解析结果，其中包含已发现的全部属性和 `complete` 标记。以下任一情况使对应 Bean 不完整：

- 不是引用类型或找不到类型声明；
- 无法枚举完整父类/接口继承链；
- 继承链中的任一类型声明缺失；
- 无法枚举方法、读取方法签名、具体化泛型或判定访问器；
- 缺失 class、泛型签名类型或其他符号解析异常。

真实且可完整枚举的零 JavaBean 属性不是解析失败；它仍然是完整结果。编译 class 失败后若源码可以完整解析，也属于完整结果。

## 结论优先级

1. 已解析属性中存在明确升级风险时保留 `RISK`。
2. 否则，Source 或 Target 任一属性模型不完整时返回 `REVIEW`。
3. 只有双方完整时才允许返回 `IGNORED` 或 `SAFE`。

即使整体是 `REVIEW`，报告仍保留已经成功解析出的属性行，避免人工审查时丢失证据。日志输出 `BEAN-ERROR reason=property-model-incomplete`。

## 验证

- 用源码 Bean 继承不存在的父类，证明旧行为会错误返回 `SAFE`。
- 修复后相同调用返回 `REVIEW`，并输出不完整日志。
- 已知风险、正常安全、缺少 setter 和 class-to-source fallback 的既有行为不回归。
